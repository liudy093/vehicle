

### 第一阶段：准备工作（环境激活与依赖）

在运行任何 ROS 2 节点之前，必须先配置当前终端的环境变量。

1. **打开容器终端**：在 VS Code 中进入 Dev Container 环境。
2. **激活环境（Source）**：必须按顺序执行以下两条命令，以加载 ROS 2 系统库和你的自定义消息接口：
```bash
source /opt/ros/foxy/setup.bash
source /home/nvidia/AutoDrive/install/setup.bash

```


3. **检查 Python 依赖**：确保安装了 MPC 求解和配置解析所需的库：
```bash
pip3 install cvxpy easydict numpy pyyaml

```



---

### 第二阶段：编译与文件校验

如果你修改了 `car_interfaces` 里的消息定义，或者第一次运行，建议先编译。

1. **进入工作空间根目录**：
```bash
cd /home/nvidia/AutoDrive

```


2. **编译消息包**：
```bash
colcon build --packages-select car_interfaces
source install/setup.bash

```


3. **检查配置文件**：确保 `src/pid/pid/config.yaml` 存在，因为代码是通过相对路径读取参数的。

---

### 第三阶段：运行代码

建议直接使用 `python3` 运行源代码，这样你可以实时修改并测试，无需反复编译控制包。

1. **运行你的集成版代码 (`lmq_mpc.py`)**：
```bash
# 推荐使用 -u 参数，防止输出缓冲区导致日志显示延迟
python3 -u src/pid/pid/new/lmq_mpc.py

```


2. **确认启动成功**：
* 如果你看到 `=== 集成版 MPC 控制器已启动 ===` 且程序没有退出（光标闪烁），说明已成功进入循环。
* **故障排除**：如果再次出现“秒退”，请确认文件末尾是否包含 `if __name__ == '__main__': main()` 入口函数。



---

### 第四阶段：功能验证与调试

由于没有真车，你需要模拟输入数据来观察 MPC 的反应。

1. **验证感知接收**（新开终端）：
```bash
# 发送模拟航速 2.0m/s 的融合数据
ros2 topic pub /fusion_data car_interfaces/msg/FusionInterface "{carspeed: 2.0, ax: 0.1, latitude: 34.585, longitude: 113.685}" -1

```


2. **验证路径接收**（新开终端）：
```bash
# 发送目标速度 3.0m/s 的规划路径
ros2 topic pub /local_path_planning_data car_interfaces/msg/LocalPathPlanningInterface "{speed: [3.0], latitude: [34.585], longitude: [113.685], angle: [0.0]}" -1

```


3. **检查数据记录**：
在左侧文件浏览器进入 `/home/nvidia/AutoDrive/src/pid/data/`，确认是否生成了新的 `.csv` 文件。

为了简化这两条繁琐的 `source` 命令，可以根据自己的使用习惯选择最合适的一种。

---

### 方案一：永久自动化（最推荐）

**适用场景**：你希望每次打开终端时，环境已经自动配置好，直接输入 `python3 ...` 就能运行。

在容器终端执行以下两条命令：

```bash
# 1. 将 ROS 2 系统底座加入自启动
echo "source /opt/ros/foxy/setup.bash" >> ~/.bashrc

# 2. 将项目工作空间加入自启动
echo "source /home/nvidia/AutoDrive/install/setup.bash" >> ~/.bashrc

# 3. 立即刷新当前环境
source ~/.bashrc

```

**实现逻辑**：`.bashrc` 是 Linux 的开机脚本。将其写入后，无论是新开 VS Code 终端还是重启容器，环境都会**自动叠加激活**。

---

### 方案二：创建快捷指令（Alias）

**适用场景**：你不想一开机就加载所有环境，但想通过一个极短的命令（如 `sr`）一键激活。

1. 执行以下命令定义别名：
```bash
echo "alias sr='source /opt/ros/foxy/setup.bash && source /home/nvidia/AutoDrive/install/setup.bash'" >> ~/.bashrc
source ~/.bashrc

```


2. **使用方法**：以后在终端只需输入两个字母 `sr` 并回车，就能完成原本两行长命令的操作。

---

### 方案三：编写一键启动脚本（Shell Script）

**适用场景**：你希望一个脚本不仅配置好环境，还能直接启动你的 MPC 控制节点。

1. 在 `/home/nvidia/AutoDrive` 目录下创建一个文件 `run_mpc.sh`：
```bash
touch run_mpc.sh
chmod +x run_mpc.sh # 赋予执行权限

```


2. 将以下内容复制进该脚本：
```bash
#!/bin/bash
# 1. 激活环境
source /opt/ros/foxy/setup.bash
source /home/nvidia/AutoDrive/install/setup.bash

# 2. 检查并提示
echo ">>> ROS 2 Platoon Environment Activated."

# 3. 运行控制代码
# 使用 -u 参数确保实时打印日志
python3 -u src/pid/pid/new/lmq_mpc.py

```


3. **使用方法**：每次只需要输入 `./run_mpc.sh`，系统会自动帮你配置环境并启动程序。
